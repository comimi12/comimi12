## 1. 프로젝트 개요

### 배경 (Pain Points)
- 현재 인포그래픽은 **모든 숫자·텍스트가 HTML에 하드코딩**되어 있다. 채용 목표(200명), 비용(14억/3.3억), 정원(10명), 매장 시뮬레이션 매출, 투자비 표 등을 바꾸려면 HTML 본문을 직접 뜯어 고쳐야 한다.
- 도넛·진행바 차트가 SVG `stroke-dasharray`/CSS `width:%`로 직접 그려져 있어, 비율 하나 바꾸려면 차트 좌표를 손으로 계산해야 한다.
- 기획 보고서라 **숫자가 자주 바뀐다**(가정·시뮬레이션 값). 매번 코드 수정은 비효율적이고 오류 위험이 크다.
- 디자인 톤, 카드 구성, 항목 추가/삭제도 코드 지식 없이는 손대기 어렵다.

### 핵심 의사결정
이 문서는 **데이터 실시간 대시보드가 아니라 전략 기획 보고서(infographic)**다. 보고서가 돕는 의사결정:
- (경영진) 조리 아카데미 모델에 **투자할지** — 일용직 14억 vs 아카데미 3.3억, 10.7억 절감 가능성
- (인사본부) 채용 3-트랙·아카데미 커리큘럼·배치 안분 정책을 **확정할지**
- (작성자=차재환) 가정 숫자를 **시나리오별로 바꿔가며** 보고서를 빠르게 갱신

### 목적
**기존 인포그래픽의 시각·구조를 100% 유지**하면서, 하드코딩된 모든 내용(숫자·텍스트·차트 비율·카드 항목)을 **코드를 건드리지 않고 편집**할 수 있는 데이터 기반 구조로 전환한다.

### 핵심 가치
1. **데이터-프레젠테이션 분리** — 내용은 데이터 파일에, 디자인은 템플릿/CSS 토큰에
2. **무손실 마이그레이션** — 현재 보이는 화면과 픽셀 단위로 동일하게 재현
3. **점진적 확장성** — 같은 데이터 구조로 ① 값만 편집 → ② 실데이터 연동 → ③ 화면 편집 UI 순으로 단계적 발전
4. **차트 자동 계산** — 비율만 입력하면 도넛/진행바 좌표는 자동 산출

## 3. 데이터 파이프라인

### 전체 흐름
```
[content.json]  --(render)-->  [템플릿 엔진]  --(출력)-->  [infographic.html]
   숫자·텍스트         차트 좌표 자동계산          현재와 동일한 화면
   차트 비율           CSS 토큰 주입
   카드 항목

(2단계) Google Sheets --(gws CLI)--> content.json  --(render)--> HTML
(3단계) 브라우저 편집모드 --(export)--> content.json
```

### 콘텐츠 데이터 모델 (content.json 스키마)

원본의 모든 블록을 1:1로 매핑한다. 최상위 구조:

| 키 | 대응 블록 | 편집 항목 |
|----|----------|-----------|
| `meta` | 헤더 eyebrow/제목/부제, footer | 부서명·제목·문서번호·날짜 |
| `headerKpis[]` | 헤더 KPI 칩 5개 | `{num, label}` |
| `pipeline[]` | 채용→아카데미→배치 3단 | `{num, label, sub, color, rows[]}` |
| `columns[]` | 3개 컬럼의 카드들 | 카드 배열 |

**카드 공통 구조**: `{ id, column, type, theme, title, body }`
- `theme`: `blue / gold / green / teal / accent` (기존 `c-*` 클래스)
- `type`별 `body` 스키마:

| type | 대응 카드 | body 스키마 |
|------|----------|-------------|
| `miniStat` | 채용 수요 현황 상단 | `stats[]: {num, label, color}` |
| `pipeBars` | 직급별 채용 난이도 | `bars[]: {label, pct, hint, color}` ← pct만 입력, 바 width 자동 |
| `donut` | 인력 수급 파이프라인 | `total, segments[]: {label, pct, count, color}` ← dasharray/offset 자동 |
| `rows` | R&R, 정책 등 | `items[]: {label, value?, color, dotColor}` |
| `org` | 조직도(채용팀/아카데미) | `nodes[]` 계층 `{level, label, style}` |
| `table` | 투자·운영비 | `columns[], rows[]: {cells[], tag?, isTotal?}` |
| `timeline` | 3개월 커리큘럼 | `items[]: {title, desc, color}` |
| `spaceGrid` | 방배동 매장 시뮬 | `cells[]: {title, desc, bg, border, color}` |
| `alerts` | 부작용·이슈, 참고사례 | `blocks[]: {kind: red/green/amber/blue/case, icon, html}` |
| `aftercare` | 사후관리 아이콘 3종 | `items[]: {icon, label, desc}` |

> 모든 카드는 `column`(1·2·3)과 배열 순서로 위치가 결정된다 → **카드 추가/삭제/순서변경이 JSON 배열 조작만으로** 가능.

### 차트 자동 계산 규칙 (편집자는 비율만 입력)
- **도넛**: `pct` 합 100 기준 → 각 세그먼트 `stroke-dasharray="{pct} {100-pct}"`, 누적 `stroke-dashoffset` 자동 산출
- **진행바**: `pct` → `width:{pct}%` 자동
- **색상**: `color` 키 하나 입력 시 그라데이션(`linear-gradient`) 자동 생성 또는 토큰 매핑

## 5. 기술 스택

> **원칙**: 무손실 재현 + 단순함 우선. 이 산출물은 Next.js 풀스택이 필요 없는 **단일 페이지 인포그래픽**이므로, 무거운 프레임워크 대신 경량 템플릿 렌더링을 채택한다.

| 기술 | 용도 | 선택 이유 |
|------|------|-----------|
| **content.json** | 콘텐츠 데이터 레이어 | 모든 숫자·텍스트·비율 분리. 중첩 구조에 적합 |
| **렌더 스크립트** (Python or 순수 JS) | JSON → HTML 생성 | 차트 좌표 자동계산, 카드 반복 렌더. 빌드 1회로 정적 HTML 출력 → 인쇄/배포 그대로 |
| **기존 CSS (`:root` 토큰)** | 디자인 | 원본 스타일 무손실 재사용 |
| **(2단계) Google Sheets + gws CLI** | 표형 데이터 외부화 | 투자비·채용트랙 표를 시트에서 편집 |
| **(3단계) contenteditable + export JS** | 화면 내 편집 모드 | 노코드 편집 — 클릭해서 수정 후 JSON 내보내기 |

> **승급 조건**: 다중 사용자 동시 편집·실시간 데이터가 필요해지면 그때 Next.js + Supabase로. 현재는 1인 편집·정적 배포라 과설계 회피.

### 구현 단계 제안
1. **Phase A (핵심)**: 원본 HTML → `content.json` + 렌더 스크립트로 분해. 출력이 원본과 픽셀 동일한지 검증
2. **Phase B**: 차트 자동계산·합계 검산 로직 추가
3. **Phase C (선택)**: 브라우저 편집 모드 또는 Sheets 연동

## 7. 누계(Cumulative) 추적 설계 ⭐

> 핵심 요구사항: 매월 실적이 **누계로 쌓이는** 운영 대시보드. 그 달 값만 입력하면 누계·진행률·절감이 자동 계산된다.

### 데이터 입력 (월 1줄 추가)
`data/data.js`의 `monthly[]` 배열에 그 달 실적을 한 줄 append. 누계는 **직접 계산하지 않는다** — 대시보드가 합산.

| 입력 항목 | 의미 | 누계 처리 |
|-----------|------|-----------|
| `채용{고급,중급,기초}` | 그 달 트랙별 채용 | Σ → 누계 채용 (목표 200 대비) |
| `입교 / 수료 / 배치` | 그 달 파이프라인 인원 | Σ → 누계 수료·배치 (목표 대비) |
| `재직중` | 그 달 말 현재 재직(시점값) | 최신값 → 재직률 = 재직/누계배치 |
| `절감액(만원)` | 그 달 일용직 대비 절감 | Σ → 누계 절감액 |

### 자동 계산·렌더 항목
1. **헤더 KPI**: 누계 채용/수료/배치 (목표 대비), 현재 재직·재직률, 누계 절감액
2. **연간 목표 대비 진행률 바**: 채용·수료·배치·재직률 (목표 대비 %)
3. **채용 트랙별 누계**: 고급/중급/기초 각 목표 대비
4. **누계 절감액 추이**: 월별 누적 막대 + 일용직 기준선·아카데미 투자 비교
5. **월별 실적 & 누계 표**: 월별 입력값 + 누계채용·누계배치·누계절감 컬럼 + 합계행

### 구현 방식
- 순수 HTML + 임베디드 JS (빌드/서버 불필요, `file://` 더블클릭 실행)
- `<script src="../data/data.js">` 로 데이터 로드 → CORS 문제 없음
- 차트(진행률 바·누계 막대)는 비율 자동 계산 — 좌표 수동 입력 제거
- 디자인 토큰(`:root`)은 원본 인포그래픽과 동일 계승

### 확장 경로
- 2단계: `data.js` → Google Sheets(gws CLI) 자동 동기화 (작성자가 시트에서 월 실적 입력)
- 3단계: 브라우저 편집 모드로 화면에서 직접 월 실적 추가
