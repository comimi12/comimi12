# 시나리오 2. 매주 월요일 임원 회의 D-1 자료 자동 작성

## 한 줄 정리

매주 일요일 저녁 8시, 다음 날 월요일 9시 임원 회의에 들어가기 전 **부문대표가 사전 검토할 회의 자료 1세트**가 자동으로 만들어져 있다.

## 임원 입장에서 지금 답답한 지점

- 본부장 4~5명이 각자 만든 슬라이드를 일요일 밤늦게 메일로 보내온다. 포맷이 다르고, 같은 숫자를 다르게 인용한다.
- 부문대표가 월요일 새벽에 슬라이드를 한 번 훑으면서 모순된 숫자 / 빠진 항목 / 수상한 청구 가설을 잡아낸다. 회의 시작 전 30분이 그 작업에 쓰인다.
- 회의에서 "지난 주에 결정한 ROAS 임계 어떻게 됐냐", "슬림쉐이크 결품 그 후 어떻게 됐냐" 같은 **이전 결정의 후속 추적**이 잘 안 된다. 본부장들도 자기 본부 안건 끝나면 다른 본부 안건은 잘 안 본다.

이 세 개를 **AI가 일요일 저녁에 미리 정리해두면**, 월요일 회의는 의사결정 30분 + 토론 30분 구조로 짧아진다.

## 시나리오 1과의 관계

| 차이 | 시나리오 1 (일일) | 시나리오 2 (주간) |
|---|---|---|
| 주기 | 매일 06:55 | 매주 일 20:00 |
| 데이터 윈도 | 어제 vs 7일 평균 | 지난 7일 vs 직전 7일 + 14일 추세 |
| 검토 시간 | 5분 | 30분 |
| 산출물 | 1페이지 알림 | 4섹션 회의 자료 |
| 사용자 | 부문대표 1명 | 부문대표 + 본부장 4~5명 |
| 데이터 소스 | 동일 4개 CSV |
| 검출 규칙 | 동일 |

→ 시나리오 1을 한 번 셋업하면 **시나리오 2는 사실상 같은 파이프라인의 다른 출력 포맷**.

## 시나리오 흐름

### 단계 0 — 환경 준비

- 시나리오 1과 동일한 4개 CSV가 매일 새벽 한 폴더로 들어와 있어야 한다.
- 추가로 **회의록 누적 파일** 1개가 필요. 매주 회의에서 결정된 액션 아이템을 부문대표가 회의 직후 한 줄씩 추가해온 파일. 이번 데모에서는 비어 있어도 됨.

권한 범위 추가 확인: 본부장 4명의 주간 슬라이드를 받아오는 슬랙 채널 / 메일 폴더 접근 권한이 부문대표에게 있는가. 없으면 단계 1을 단순화한다(본부장 슬라이드 통합은 다음 시즌으로).

### 단계 1 — 일요일 저녁 자동 트리거

부문대표가 직접 입력하지 않는다. 일요일 20:00에 자동 시작.

```
{매주 일 20:00} → AI: "주간 임원 회의 자료 만들고 김호균 대표에게 카톡 + 메일로 알림"
```

전제: 자동 트리거이지만 실제 발송 직전에 부문대표 검토 단계가 끼어든다(단계 5).

### 단계 2 — AI가 데이터 통합 (주간 단위)

- 지난 7일 (4/30 ~ 5/6) 매출 / 광고 / 재고 / 고객 지표
- 직전 7일 (4/23 ~ 4/29) 대비 증감 + 14일 전체 추세선
- 채널별 / SKU별 / 본부별 분리
- **이전 회의록**을 읽어 직전 회의에서 정한 액션 아이템 후속 상태 자동 추적

### 단계 3 — 이상 신호 + 의사결정 후보 분리

시나리오 1의 검출 규칙(재고 7일 / ROAS -15% / 매출 ±25% / 결품)을 그대로 사용하되, 출력 포맷이 다름:

- **확정 보고**: 평소대로 굴러간 항목 (정상 신호) — 표 1줄씩만
- **알림**: 빨강·노랑 신호 — 본부장이 답해야 할 항목으로 분리
- **의사결정 후보**: 데이터에서 발견된, 임원 회의에서 결정해야 진행 가능한 항목 (예: "쿠팡 ROAS 임계 2.8 이하 진입 → 광고 운영 재설계 / 임계 재정의 중 택 1")
- **이전 결정 추적**: 직전 주에 정한 액션 아이템의 진행 상태 (회의록 기반)

### 단계 4 — 4섹션 회의 자료 생성

산출물: `reports/2026-05-04_weekly-exec-meeting.md` (이번 데모는 5/4 회의 가정으로 만들어둠)

구성:

1. **이번 주 한 줄** — 부문 매출 / 7일 추세 / 가장 큰 변화 1줄
2. **본부별 정상 진행** — 표 1개. 4~5행. 빨강 없는 본부는 여기까지만
3. **결정이 필요한 안건** — 3개 이내. 각 안건마다 (배경 / 데이터 / 선택지 2~3개 / 부문대표 의견 자리)
4. **이전 결정 추적** — 직전 회의에서 정한 액션 아이템의 진행 상태

길이 제약: PDF 변환 시 A4 4장 이내. 이미지 0개 (마크다운 표만).

### 단계 5 — 사람 확인 (부문대표)

일요일 21:00 ~ 월요일 08:30 사이 부문대표가 30분 검토. 동일 세션에서 추가 질문/수정:

- "결정 안건 ②번에 본부장 A 의견 자리만 비워두지 말고, 지난 회의록에서 그 본부장이 비슷한 주제에 어떤 입장이었는지 한 줄 채워줘."
- "이번 주 정상 진행 항목 중 마케팅본부 행에 있는 ROAS는 평균이 아니라 채널별 가중평균으로 다시 계산해줘."

부문대표가 추가/수정한 항목은 회의록에 자동 반영(단계 6 이후 다음 주에 후속 추적될 수 있도록).

### 단계 6 — 본부장 4명에게 사전 공유

월요일 08:30, 부문대표가 "이대로 본부장 4명에게 사전 공유"라고 한 줄 입력. AI가 슬라이드용 본문 + 슬랙 메시지 본문 출력. **실제 발송은 사람이 누른다**(자동 발송 안 함).

### 단계 7 — 월요일 회의 후 액션 아이템 추가

회의 직후 부문대표가 한 줄씩 추가:

```
ACTION 2026-05-04: 마케팅본부장 / 쿠팡 ROAS 광고 운영 재설계 1차 안 / 5/9까지
ACTION 2026-05-04: 생산본부장 / 슬림쉐이크 입고 사이클 검토 / 5/11까지
```

이 액션 아이템이 다음 주 일요일 단계 3에서 자동 추적된다(루프 닫힘).

## 산출물 미리보기

`reports/2026-05-04_weekly-exec-meeting.md`에 같이 만들어둠. 시나리오 단계 4의 결과물.

## 이 시나리오가 임원진에게 강한 이유

- **임원 본인이 검토자다**: 본부장이 만든 자료를 정합성 검증하느라 새벽에 시간 쓰는 구조에서, AI가 통합·정합성·이전 결정 추적까지 미리 해두고 임원은 의견만 단다.
- **루프가 닫힌다**: 회의 → 액션 아이템 → 다음 주 자동 추적. 본부장이 액션 미이행이어도 임원이 그것만은 매주 알게 된다.
- **시나리오 1과 같은 데이터 인프라**: 도입 비용이 한 번만 든다. 일일 + 주간이 같은 파이프라인에서 나온다.
- **본부장 부담이 줄어든다**: 본부장이 슬라이드를 직접 만들지 않아도 부문대표가 의사결정 가능한 자료가 생긴다. 본부장은 의견 + 답변만 준비한다.

## 단계 0에서 막힐 가능성

| 위험 | 우회 |
|---|---|
| 회의록 누적 파일 부재 | 첫 4주는 회의 직후 부문대표가 직접 입력 (한 줄/안건) |
| 본부별 분리 데이터 부재 | 첫 시즌은 채널별/SKU별만. 본부별 매핑은 본부장 협의 후 |
| 직전 회의 액션 아이템 추적 정확도 | 첫 4주는 AI가 후보만 띄우고 부문대표가 ✓/✗ 표시. 5주차부터 신뢰 시작 |

6. 본부장 시점으로 확장 가능성 토의 (10분)
