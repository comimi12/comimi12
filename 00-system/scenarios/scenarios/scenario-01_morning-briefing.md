# 시나리오 1. 매일 아침 7시 헬스케어 부문 임원 브리핑

## 한 줄 정리

매일 아침 7시, 부문대표가 폰을 열기 전에 어제 매출·광고·재고가 1페이지로 정리되어 있고, 어제 일어난 이상 신호 3개가 강조되어 있다.

## 임원 입장에서 지금 답답한 지점

- 채널이 6개(자사몰·쿠팡·마켓컬리·편의점·무신사·기타). 채널별 매출은 매일 다른 시스템에 흩어져 있고, 본부장이 매일 아침 8~9시 사이에 메일로 보내주는 한 장으로 받는 구조.
- 광고 ROAS는 매주 회의 자료에서 본다. 일별로 보지 않는다. 떨어지고 있는 건 알지만, 어느 채널에서 며칠째 떨어지고 있는지 즉답이 안 된다.
- 재고 결품은 사후에 듣는다. "그거 5월 4일에 자사몰 결품 났어요"를 5월 6일에 듣는 식.

이 세 개가 **부문대표가 아침에 폰을 열기 전에 정돈되어 있다면**, 9시 회의 들어가기 전 30분이 비교 분석에 쓰인다.

## 시나리오 흐름

### 단계 0 — 환경 준비

다음 4개 데이터 소스가 매일 새벽 자동으로 한 폴더에 들어와 있어야 한다. 오늘 데모에서는 샘플 CSV로 대체.

| 소스 | 파일 | 비고 |
|---|---|---|
| 채널별 일 매출 | `daily_channel_sales.csv` | 채널 6개 × 14일 |
| SKU별 매출/재고 | `sku_performance.csv` | 주력 5개 SKU |
| 광고 ROAS | `ad_roas.csv` | 4개 매체 × 14일 |
| 고객 지표 (주간) | `customer_metrics.csv` | 채널별 신규/재구매율 |

권한 범위 확인 필요: 부문대표 단위로 채널별 광고 플랫폼 / ERP 재고 데이터에 read 권한이 있는가. 없으면 정보보안 / 운영팀 협의가 단계 0의 실제 작업이 됨.

### 단계 1 — 부문대표가 한 줄 요청

> "오늘 아침 부문 브리핑 만들어줘. 어제 vs 7일 평균. 이상 신호 강조."

전제: AI는 단계 0의 4개 CSV 경로를 미리 알고 있고, "어제"는 매일 자동으로 직전일로 해석.

### 단계 2 — AI가 데이터 통합

- 어제(2026-05-07) 매출 합계: 채널별 + 전체
- 7일 평균(2026-04-30 ~ 2026-05-06) 대비 증감률
- 광고 ROAS 어제 / 7일 평균 / 최근 7일 추세 기울기
- SKU별 어제 판매량 + 잔여 재고일수

### 단계 3 — 이상 신호 자동 검출

검출 규칙(이번 데모 기준):

- **재고 위험**: `days_of_stock < 5` → 빨강. `< 7` → 노랑.
- **ROAS 이탈**: 채널별 어제 ROAS가 7일 평균 대비 -15% 이하 → 빨강.
- **매출 급변**: 채널별 어제 매출이 7일 평균 대비 ±25% 이상 → 노랑.
- **결품 발생**: 어제 판매량 0이면서 직전 7일 평균 판매량 > 0 → 빨강.

이 규칙은 부문대표가 처음에 한 번 같이 정해두면, 이후 AI가 동일 규칙으로 매일 검출.

### 단계 4 — 1페이지 보고서 생성

산출물: `reports/2026-05-07_briefing.md`

구성:
1. 헤더 — 어제 총매출 + 전7일 대비 증감
2. 채널별 표 (6행)
3. SKU 알림 — 결품/임박 SKU만
4. 광고 ROAS 알림 — 이탈 채널만
5. 임원 의사결정 필요 항목 (3개 이내, 행동 단위)

길이 제약: 휴대폰 한 화면. 이미지·차트 없이 텍스트만.

### 단계 5 — 사람 확인

부문대표가 5분 내 검토. 의문이 있으면 AI에게 같은 세션에서 추가 질문:

- "쿠팡 ROAS 왜 떨어진 거야?"
- "슬림쉐이크 결품 언제부터야?"
- "자사몰 5월 1일~3일 매출은 사전예약 효과 맞아?"

AI는 동일 4개 CSV 안에서 답하고, **데이터에 없으면 모른다고 말한다**. 외부 추정 금지.

### 단계 6 — 임원진 공유 (수동 트리거)

부문대표가 검토를 마치면 한 번 더 요청:

> "이대로 이정호·정정민 본부장에게 슬랙 DM으로 보내줘."

AI는 마크다운을 슬랙용으로 정리해 보낼 수 있는 본문을 출력. 실제 발송 버튼은 사람이 누른다(자동 발송 안 함).

## 이 시나리오가 임원진에게 강한 이유

- **실무자 시나리오가 아니다**: 사장이 직접 효익을 본다. 본부장에게 "아침 보고 안 만들어도 된다"가 부수 효과.
- **새로운 시스템이 아니다**: 데이터가 매일 한 폴더로 떨어지면 끝. 통합 BI는 따로 안 사도 된다.
- **검출 규칙이 사람 손에 있다**: 임원이 처음 한 번 규칙을 정한다. AI가 임의 판단하지 않는다.
- **확장 경로가 명확하다**: 같은 패턴을 본부장 시점(이정호 신사업 / 정정민 피플)으로 복제 가능.

## 단계 0에서 막힐 가능성

| 위험 | 우회 |
|---|---|
| 채널 데이터 권한 미확보 | 첫 4주는 본부장이 보내주는 메일 첨부 CSV를 한 폴더에 옮기기만 자동화 |
| ERP 재고 시스템 API 없음 | 매일 새벽 CSV export → 같은 폴더에 떨굼 |
| 광고 플랫폼 4개 통합 | 첫 단계는 쿠팡·메타 2개만. 나머지는 2주 후 |

## 데모 시나리오 진행 순서 (45~60분 가정)

1. 단계 0~3을 맥락 설명 (10분)
2. 단계 1을 임원이 직접 입력 (5분)
3. 단계 4 보고서가 화면에 뜨는 걸 함께 본다 (5분)
4. 단계 5의 추가 질문을 임원이 1~2개 던져본다 (10분)
5. 같은 패턴이 어떻게 본부장 단위 / 영업 단위로 확장되는지 토의 (15~20분)
