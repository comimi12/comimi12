# secure-doc-intake — 보안문서 정식 반출분 자동 정리

## 이 스킬이 하지 않는 것 (하드 경계, 협상 불가)

**이 스킬은 어떤 형태의 암호화 해제·DRM 우회·보호 로직 무력화도 수행하지 않는다.** FASSO 같은
문서보안 프로그램은 조직이 승인하지 않은 프로그램의 문서 접근을 막으려고 존재하는 통제 장치다.
이 스킬이 다루는 대상은 **사용자가 이미 정식 권한으로, 공식 클라이언트의 내보내기/다른 이름으로
저장 기능을 통해 평문화한 파일**뿐이다. 그 전 단계(보호된 원본을 여는 것)는 사람이 공식 클라이언트로
직접 해야 한다 — 이 스킬도, 다른 어떤 스킬도 그 단계를 대신하지 않는다.

Step 1의 무결성 게이트를 통과하지 못한 파일(여전히 암호화·손상 의심)은 **절대 열거나 변환하지
않고 그대로 둔 채 보고만 한다.** 이 게이트는 "묻지 말고 자동으로" 요청과 무관하게 항상 실행된다 —
자동화 대상은 "이미 평문인 파일의 정리 작업"이지 "판정 자체를 생략하는 것"이 아니다.

---

## Step 0: 인박스 폴더 확인/생성

```bash
mkdir -p 00-inbox/secure-export
```

인자로 특정 파일 경로가 주어지면 그 파일만 대상. 없으면 `00-inbox/secure-export/` 전체 스캔
(Glob `00-inbox/secure-export/**/*`). 대상 0건이면 "반입 대기 파일 없음" 보고 후 종료.

## Step 1: 무결성 게이트 (필수, 스킵 불가)

```bash
python .claude/skills/secure-doc-intake/scripts/check_integrity.py 00-inbox/secure-export
```

- `OK` → Step 2 진행.
- `SUSPECT` → 그 파일은 **건드리지 않는다.** "이 파일은 여전히 보호 상태이거나 손상된 것으로 보입니다.
  FASSO 공식 클라이언트에서 문서를 열고 '내보내기' 또는 '다른 이름으로 저장'으로 다시 반출해 주세요"
  라고 보고. 다른 OK 파일들 처리는 계속 진행(한 파일의 실패가 전체를 막지 않음).
- `UNKNOWN`(미지원 확장자) → 자동 판정 불가. 파일을 열지 않고 사용자에게 형식을 보고, 텍스트로
  안전하게 읽을 수 있는 형식(예: `.txt`, `.md`, `.csv`)이면 Read로 직접 확인 후 진행, 아니면 보류.

## Step 2: 형식별 변환 라우팅

무결성 게이트를 통과한 파일만 확장자 기준으로 라우팅. 확인 질문 없이 바로 위임한다.

| 확장자 | 처리 |
|--------|------|
| `.xlsx` `.xls` | `Skill excel-to-csv` 호출 |
| `.pdf` | `Skill pdf-to-md` 호출 |
| `.hwp` `.hwpx` | `Skill hwpxskill` 호출 |
| `.csv` | 인코딩만 확인(UTF-8 아니면 `excel-to-csv --encoding` 활용), 필요시 `Skill csv-clean` 제안 |
| `.txt` `.md` | 변환 불필요, Step 3으로 |
| 그 외(이미지 등) | 변환 불필요, 원본 그대로 Step 3으로 |

## Step 3: 정리 (자동, 목적지 추론)

각 파일(원본 + 변환 결과물)을 다음 우선순위로 배치한다:

1. 파일명·내용에서 기존 프로젝트(`10-projects/*`)와 명확히 매칭되면 그 프로젝트 폴더.
2. 재무/손익/매출 등 반복 운영 데이터로 보이면 `20-operations/` 해당 폴더.
3. 매칭 불명확하면 `00-inbox/` 루트(추후 주간 정리에서 재배치 — 기존 인박스 규칙과 동일).

원본 파일은 처리 완료 후 `50-resources/attachments/secure-imports/`로 이동(첨부파일 보관 규칙 준수).
`00-inbox/secure-export/`는 다음 반입을 위해 비워둔다.

## Step 4: 결과 보고

```
=== secure-doc-intake 처리 결과 ===
OK 처리: N건
  - <파일명> → <변환 결과 경로> (원본: 50-resources/attachments/secure-imports/)
SUSPECT (미처리, 재반출 필요): M건
  - <파일명> — <사유>
UNKNOWN (수동 확인 필요): K건
  - <파일명>
```

## 알려진 한계

- 무결성 게이트는 확장자-매직바이트/ZIP구조 정합성만 본다. FASSO가 평문 확장자를 유지한 채
  내용만 손상 없이 위변조하는 극단적 케이스는 감지 못할 수 있음 — 의심스러우면 Read로 내용 샘플을
  직접 확인.
- `UNKNOWN` 확장자는 자동 판정 로직이 없어 보수적으로 보류한다. 새 형식이 반복되면
  `scripts/check_integrity.py`의 `MAGIC`/`ZIP_BASED`/`TEXT_BASED`에 추가.
- 이 스킬은 **반출 이후** 단계만 자동화한다. 반출 자체(보호된 원본을 공식 클라이언트로 여는 것)는
  항상 사람이 수행.
