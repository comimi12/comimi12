# 시작하기 — 신규 설치 안내 (Windows)

> 비개발자도 따라 할 수 있도록 처음부터 끝까지 정리했습니다.
> 명령어는 모두 **터미널(Win+X → 터미널)** 에 한 줄씩 붙여넣고 Enter 치면 됩니다.

---

## 0단계 — 준비물 설치 (처음 한 번만)

clone 명령어가 작동하려면 **Git**이 필요합니다.

1. **Git 설치** → https://git-scm.com/download/win
   - 다운로드한 파일 실행 후, 별다른 선택 없이 계속 **"Next(다음)"** 만 누르면 됩니다.
2. **Claude Code 설치** (이미 있으면 생략)
3. ⚠️ 설치가 끝나면 **터미널을 새로 열어야** `git` 명령이 인식됩니다.

설치가 잘 됐는지 확인하려면 터미널에 입력:
```powershell
git --version
```
→ `git version 2.xx.x` 처럼 버전이 나오면 성공.

---

## 1단계 — 터미널 열기

키보드에서 **Win + X** → 메뉴에서 **"터미널"** 클릭
(Windows 11 기본. Windows 10이면 "Windows PowerShell")

---

## 2단계 — 받을 위치로 이동

clone하면 **지금 있는 폴더 안에** `comimi12` 폴더가 새로 생깁니다.
원하는 위치로 먼저 이동하세요. (그냥 사용자 폴더에 받을 거면 이 단계는 건너뛰어도 됩니다.)

```powershell
cd C:\Users\내이름
```
> `내이름` 부분을 본인 Windows 사용자 폴더 이름으로 바꾸세요.

---

## 3단계 — 워크스페이스 내려받기 (clone)

```powershell
git clone https://github.com/comimi12/comimi12.git
```

→ `comimi12` 폴더가 생기고, 모든 파일·폴더가 다운로드됩니다.

---

## 4단계 — Claude Code로 열기

```powershell
cd comimi12
claude
```

---

## 5단계 — 초기 설정 (자동)

Claude Code가 열리면, 그냥 이렇게 입력하세요:

```
워크스페이스 세팅해줘
```

`setup-workspace` 스킬이 대화형으로 아래를 자동 진행합니다:

1. **프로필 작성** — 이름·역할·관심사 등 4개 질문
2. **Python 환경** — 데이터 분석 스킬용 `.venv` 생성 (원할 때만)
3. **도구 체크** — `git`, `gws` 설치 상태 확인
4. **첫 Daily Note** — 오늘의 기록 시작

각 단계는 건너뛸 수 있고, 이미 된 항목은 자동으로 넘어갑니다.

## 자주 막히는 곳

| 증상 | 해결 |
|------|------|
| `git`을 찾을 수 없다고 나옴 | Git 설치 후 **터미널을 새로 열기** |
| `claude`를 찾을 수 없다고 나옴 | Claude Code 설치 확인 후 터미널 새로 열기 |
| clone이 권한 오류로 막힘 | 비공개 저장소면 GitHub 로그인 필요 — 관리자에게 문의 |
| 폴더가 어디 생겼는지 모름 | 3단계에서 `cd`로 이동한 위치 안의 `comimi12` 폴더 |
